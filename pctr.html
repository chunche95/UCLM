Conceptos básicos.

En este capítulo se plantean los aspectos básicos relativos al concepto de proceso, estableciendo las principales diferencias con respecto a una hebra o hilo y haciendo especial hincapié en su creación y gestión mediante primitivas POSIX.
La problemática que se pretende abordar está vinculada al concepto de sistema operativo multiproceso, donde se mejora la eficiencia del sistema pero plantea un problema con la sincronización en el acceso a los recursos. Típicamente, esta problemática no se resuelve a nivel de sistema operativo, siendo responsabilidad del programador el garantizar un acceso consistente a los recursos.
Este planteamiento general, se introduce mediante el problema del productor-consumidor, teniendo que compartir un buffer, y da lugar al concepto de sección crítica. Entre las soluciones planteadas destada el uso de semáforos y el paso de mensajes como mecanismos cásicos de sincronización.

El concepto de proceso

Un proceso se puede definir como un programa en ejecución. Además del propio código al que está vinculado, un proceso incluye el valor de un contador de programa y el contenido de ciertos registros del procesador. Generalmente, un proceso también incluye la pila del proceso, usada para almacenar datos temporales, como variables locales, direcciones de retorno y parámetros de funciones, y una sección de datos con variables globales.
Finalmente, un proceso también puede tener una sección de memoria reservada de manera dinámica.

Gestión básica de procesos

Cada estado se define en base a la actividad desarrollada por un proceso en un instante determinado:

  1. 1. Nuevo donde el proceso está siendo creado.
  2. 2. En ejecución donde el proceso está ejecutando operaciones o instrucciones.
  3. 3. En espera donde el proceso está a la espera de que se produzca un determinado eventom típicamente la finalización de una operación de E/S.
  4. 4. Preparado donde el proceso está a la espera de que le asignen alguna unidad de procesamiento.
  5. 5. Terminado donde el proceso ya ha finalizado su ejecución.
En UNIX, los procesos se identifican con un ID del proceso. El proceso encargado de ejecutar la solicitud para crear un proceso se denomina proceso padre, mientras que el nuevo proceso se denomina proceso hijo. Las primitivas POSIX usadas para obtener dichos identificadires son los siguientes:

Primitivas POSIX ID Proceso. 

          
            #include 
            #include 

            pid_t getpid (void);   // ID proceso.
            pid_t getppid (void);  // ID proceso padre.

            uid_t getuid (void); // ID usuario.
            uid_t geteuid (void); // ID usuario efectivo.
Las primitivas getpid() y getppid() se usa para obtener los identificadores de un proceso, ya sea el suyo propio o el de su padre.
UNIX asocia cada proceso a un usuario en particular, propietario del proceso.
Proceso init

init es el primer proceso en ejecución, con PID 1m y es el responsable de la creación del resto de procesos.

Para obtener el ID del usuario se hace uso de getuid().
Cada usuario tiene asociado un ID único dentro del sistema, es posible usar la primitiva geteuid() para obtener el ID del usuario efectivo.

Primitivas básicas en POSIX

La creación de procesos se realiza mediante la llamada al sistema fork(). Cuando se hace esto, se genera una copia exacta que deriva en un nuevo proceso, el proceso hijo, que recibe una copia del espacio de direcciones del proceso padre. Ahora, ambos procesos continúan su ejecución.

Llamada fork() al sistema. 

              
              #include 
              #include 

              pid_t fork(void);
fork() devuelve el valor 0 al hijo y el PID del hijo al padre. La creación de dos procesos totalmente idénticos no sería muy útil.

Independencia

Los procesos son independientes entre sí, por lo que no tienen mecanismos implícitos para compartir información y sincronizarse. Incluso con la llamada fork(), los procesos padre e hijo son totalmente independientes.

              int main(void) {
                int *valor = malloc(sizeof(int));
                *valor = 0;
                fork();
                *valor = 13;
                printf(%ld: %d\n", (long)getpid(), *valor);

                free(valor);
                return 0;
